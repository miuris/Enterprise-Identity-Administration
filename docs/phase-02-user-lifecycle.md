# Phase 2 – User Lifecycle Management

## 🎯 Objective

Implement user lifecycle management by provisioning organizational identities, assigning Microsoft 365 licenses, and validating user account configuration within Microsoft Entra ID.

---

## 🏢 Business Requirement

As Kimble Glass expands, Human Resources must be able to onboard new employees quickly while ensuring every user receives the appropriate identity, licensing, and access to Microsoft 365 services.

The objective of this phase is to establish a repeatable user provisioning process that supports secure identity management and future access governance initiatives.

---

## ⚙️ Implementation

Completed the following tasks:

- Created a new employee account
- Configured user profile information
- Assigned Microsoft 365 Business Premium license
- Verified account activation
- Reviewed assigned services
- Validated successful provisioning in Microsoft Entra ID
- Confirmed user appears in Microsoft 365 Admin Center

## 🤔 What Surprised Me

I expected creating a user to simply involve entering a name and email address. Instead, I learned that user provisioning is closely tied to licensing, service availability, and identity management.

Assigning a Microsoft 365 Business Premium license immediately determines which Microsoft services the user can access, making licensing an important part of the identity lifecycle rather than a separate administrative task.
